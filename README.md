# 1099TaxCalcs.com — Phase 4

A static, privacy-conscious calculator website built with HTML, shared CSS, and vanilla JavaScript for GitHub Pages. Financial entries remain in the browser: there is no backend, account, storage, analytics, tracking, or external calculation API.

Phase 4 adds six calculators—federal income-tax planning, Schedule C profit, contractor hourly rate, business-expense deduction planning, 1099 tax reserve, and freelance invoice income—plus eight supporting guides. See `PROJECT-INVENTORY.md`.

## Project structure

- `index.html`: homepage and simplified quick estimator
- `calculators/`: directory page and six calculator pages
- `data/tax-years/2026.js`: canonical 2026 federal values, effective periods, dates, verification metadata, and official sources
- `assets/js/tax-config.js`: browser compatibility facade for the canonical year data
- `assets/js/calculator-core.js`: shared validation, formatting, and tax formulas
- `assets/js/calculators.js`: Phase 2 calculator controllers
- `assets/js/phase3-calculators.js`: split mileage, payment schedule, safe-harbor education, transparency, and source panels
- `assets/js/calculators/`: shared Expansion Pack controller plus one controller per Phase 4 calculator
- `build_phase4.mjs`: source-of-truth generator for the six Phase 4 calculators, eight guides, directories, homepage expansion, sitemap, and inventory
- `assets/css/styles.css`: shared responsive and print styles
- `scripts/`: dependency-free configuration, calculator, and site validators
- `tests/browser-regression.html`: same-origin browser regression harness
- `ANNUAL-TAX-REVIEW.md`: next-year update checklist
- `TESTING.md`: automated and browser test procedure

## Source versus generated files

Public HTML files are the deployed pages and are Phase 3 source files. `build_phase1.mjs` is the legacy Phase 1 generator. Its `phase2Managed` preservation list prevents it from overwriting current calculator entry points, shared assets, year data, documentation, and tests. Informational pages remain legacy-generated unless listed as managed. Always review the diff after regeneration.

Phase 4 public pages are generated by `build_phase4.mjs`. Edit its page data/templates and the corresponding calculator controller, then run `node build_phase4.mjs`. Existing Phase 1–3 calculator pages remain preserved.

## Preview locally

Run `python -m http.server 8000` from this directory and visit `http://localhost:8000/`. Use HTTP rather than `file:///`; it matches GitHub Pages/custom-domain URL behavior and enables the browser regression harness.

## Regenerate safely

Make a backup or use a clean reviewable working tree, run `node build_phase1.mjs`, and inspect every changed file. Never delete a managed file merely to force legacy output. Keep generator preservation entries synchronized when adding a Phase 3 source file.

## Validation and tests

Run:

```text
node scripts/validate-tax-config.mjs
node scripts/test-calculators.mjs
node scripts/validate-site.mjs
node build_phase4.mjs
```

Then follow the browser procedure in `TESTING.md`. These scripts use only Node built-ins and do not access external APIs.

## Updating to a new tax year

Follow `ANNUAL-TAX-REVIEW.md`. Verify sources directly on IRS.gov or SSA.gov, create a new canonical year file, update browser references and visible labels, then run all validation and browser checks. Values without official support remain user-editable and labeled as requiring verification.

For a midyear rate change, add separate records with exact, contiguous effective dates. Never average the periods or silently apply the later rate to the entire year. The 2026 business mileage configuration uses 72.5¢ through June 30 and 76¢ beginning July 1.

## CNAME and GitHub Pages safety

Preserve `CNAME` byte-for-byte and never generate it from either build script. The Phase 4 audit found the current file empty, so domain configuration requires owner review before upload. Before uploading to GitHub Pages, review changes, run all checks, and upload only intended public/source files. This workflow does not commit, push, publish, or alter repository settings automatically.

Do not publish temporary browser profiles, `.edge-smoke*`, local test output, screenshots, logs, OS metadata, or editor temporary files. The `.gitignore` records these exclusions without ignoring public website assets.

## Official-source review

The canonical year file records source titles, URLs, effective dates, review notes, verification status, and the last-reviewed date. Review the source document itself; a URL being present is not a substitute for checking the applicable year and later corrections or announcements.
